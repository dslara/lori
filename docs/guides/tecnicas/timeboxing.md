## 🎯 O que é Timeboxing? 
**Timeboxing** é alocar um bloco fixo de tempo para uma atividade específica. Quando o tempo acaba, você para, independente de ter terminado ou não.

### Princípio Fundamental
```
"O trabalho se expande para preencher o tempo disponível"
                              - Lei de Parkinson

Solução: Limitar o tempo força foco e eficiência.
```

## 🧠 Quando Usar

### ✅ USE para:
- Tarefas com escopo definido
- Evitar perfeccionismo excessivo
- Projetos complexos (dividindo em caixas)
- Reuniões e calls
- Dias com muitas tarefas diferentes

### ❌ NÃO USE para:
- Tarefas criativas que precisam de fluxo contínuo
- Problemas de debugging complexos (use até resolver)

## ⚖️ Timeboxing vs Pomodoro

| Aspecto | Timeboxing | Pomodoro |
|---------|------------|----------|
| **Duração** | Variável (15-120 min) | Fixa (25 min) |
| **Pausa** | Ao final da tarefa | A cada 25 min |
| **Foco** | Tarefa específica | Sessão de trabalho |
| **Melhor para** | Projetos, tarefas definidas | Estudo contínuo |

### Quando usar cada um?
```
POMODORO: Estudo, leitura, prática de código
TIMEBOXING: Projeto com deadline, reunião, escrita
```

## 🛠️ Como Implementar

### Passo 1: Estimar Tarefa
```
Tarefa: "Implementar função de login"
Estimativa: 90 minutos (inclui testes)
```

### Passo 2: Definir Timebox
```
Timebox: 90 minutos
Timer configurado
MODO FOCO ATIVADO
```

### Passo 3: Executar
- Trabalhar na tarefa até o timer tocar
- Não perfeccionar além do necessário
- Se terminar antes: revisar ou próxima tarefa

### Passo 4: Revisar
```
Terminou no tempo?
├─ Sim → Estimativa boa, próxima tarefa
└─ Não → Tarefa maior que pensava, dividir na próxima
```

## 📝 Exemplos

### Dia de Trabalho
```
09:00-10:30 (90min) → Implementar feature X
10:30-10:45 (15min) → Pausa
10:45-12:00 (75min) → Code review
12:00-13:00 (60min) → Almoço
13:00-14:30 (90min) → Documentação
```

### Sessão de Estudo
```
Bloco 1: 60 min → Ler capítulo de livro
Bloco 2: 45 min → Fazer exercícios
Bloco 3: 30 min → Revisar com quiz
```

### Reuniões
```
Daily: 15 min (em pé, cronometrado)
Planning: 60 min (dividido em tópicos de 10 min)
Retrospective: 45 min
```

## 💡 Dicas

### Regra do 80%
Se terminar em 80% do tempo, está bom. Não perfeccionar os 20% finais.

### Timebox de Proteção
Reserve blocos de tempo para:
- Email/Slack (30 min, 2x ao dia)
- Tarefas inesperadas (1h reserva)
- Estudo pessoal (1h por dia)
