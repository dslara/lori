## 🎯 O que é Environment Design? 
### Definição
**Environment Design** é o ato intencional de **configurar seu ambiente físico e digital** para eliminar distrações e facilitar o foco profundo.

### Princípio Central
```
❌ Abordagem comum: "Vou resistir às distrações"
   → Falha: Força de vontade é finita

✅ Abordagem correta: "Vou remover as distrações"
   → Sucesso: Ambiente faz o trabalho por você
```

### 🔗 Relação com James Clear (Atomic Habits)
Esta técnica implementa **Cue-Routine-Reward**:
- **Cue**: Ambiente preparado (estímulo visual/físico)
- **Routine**: Comandos `/ul-study-start` → `/ul-study-start` → `/ul-study-end`
- **Reward**: Streak atualizado, progresso visual

## 💡 Por que usar?

### A Ciência da Distração

| Fator | Impacto | Solução via Environment Design |
|-------|---------|-------------------------------|
| **Attention Residue** | Trocar tarefas deixa "resíduo" na mente | Ambiente para 1 tarefa |
| **Dopamina** | Notificações geram vício químico | Eliminar notificações |
| **Decision Fatigue** | Cada decisão gasta energia | Rotinas automáticas |
| **Visual Clutter** | Bagunça visual aumenta cortisol | Mesa minimalista |

### Evidência Científica

- ✅ **Estudo de Stanford**: Ambiente organizado aumenta foco em 40%
- ✅ **Pesquisa de notificações**: Celular na mesa reduz QI em 10 pontos (mesmo desligado!)
- ✅ **Clutter research**: Bagunça visual aumenta cortisol (hormônio de stress)

## 🧠 Quando Usar

### ✅ USE Environment Design

| Situação | Ação |
|----------|------|
| **Setup inicial de estudo** | Configurar ambiente antes de começar |
| **Mudança de local** | Adaptar novo espaço para foco |
| **Queda de produtividade** | Revisar e otimizar ambiente |
| **Início de módulo importante** | Upgrade no setup |
| **Após falha de foco** | Identificar gatilhos no ambiente |

### ❌ NÃO Ignore

| Situação | Consequência |
|----------|--------------|
| **Celular na mesa** | 47% mais distrações (mesmo sem tocar!) |
| **Múltiplas abas** | Attention residue entre tarefas |
| **Mesa bagunçada** | Stress visual, dificuldade para começar |
| **Notificações ligadas** | Interrupção constante de foco profundo |

## 🛠️ Como Usar (Passo a Passo)

### Setup Físico (Ambiente Real)

#### Passo 1: Escolher Local de Estudo

```markdown
# ✅ IDEAL: Espaco dedicado
- Mesa específica para estudo
- Cadeira ergonômica
- Iluminação boa (luz natural ou LED)
- Ventilação adequada
- Possibilidade de fechar porta

# ⚠️ ACEITÁVEL: Espaco compartilhado
- Biblioteca
- Coworking
- Canto de casa com divisória visual

# ❌ EVITAR:
- Cama (associação com sono)
- Sofá (associação com descanso)
- Área de convivência (interrupções)
```

#### Passo 2: Configurar Mesa (5 min)

```markdown
# CHECKLIST DE MESA MINIMALISTA

## ✅ PODE TER na mesa:
- [ ] Computador (só!)
- [ ] 1 copo de água
- [ ] Caderno + caneta (para anotações rápidas)
- [ ] Fones de ouvido (opcional)
- [ ] Timer (Pomodoro)

## ❌ NÃO PODE TER na mesa:
- [ ] Celular
- [ ] Comida
- [ ] Objetos não relacionados
- [ ] Papéis/pastas de outras tarefas
- [ ] Brinquedos/fidgets (durante foco)
```

**Regra de ouro**: Se você não usa naquela sessão, não fica na mesa.

#### Passo 3: Controle de Interrupções Físicas

```markdown
# SINALIZAÇÃO "NÃO PERTURBE"

## Opção 1: Porta fechada
- Fechar porta antes de começar
- Colocar aviso: "Estudando até [hora]"

## Opção 2: Fones de ouvido
- Fones = sinal universal de "não perturbe"
- Pode ser sem música (só o visual já funciona)

## Opção 3: Comunicação com familiares/colegas
- "Vou estudar das 10h às 12h"
- "Só me chamar se for emergência"
- Definir o que é emergência

## Backup: Lista "Pensar Depois"
- Papel ao lado para anotações rápidas
- Se lembrar de algo: anota e volta ao foco
- Lidar COM DEPOIS, não durante
```

### Setup Digital (Ambiente Virtual)

#### Passo 4: Configurar Computador (5 min)

```bash
# CHECKLIST DIGITAL

## Navegador
- [ ] Fechar TODAS as abas (Ctrl+Shift+W ou Cmd+Shift+W)
- [ ] Abrir só o necessário (docs, IDE)
- [ ] Desativar notificações do navegador
- [ ] Ativar modo "Do Not Disturb" do OS

## Desktop
- [ ] Fechar apps desnecessários
- [ ] Esconder ícones da área de trabalho
- [ ] Wallpaper neutro (não distrai)
- [ ] Modo escuro (menos fadiga visual)

## IDE/Editor
- [ ] Modo "Zen" ou "Distraction-free" (esconde menus)
- [ ] Fonte grande o suficiente (14-16px)
- [ ] Tema de alto contraste (fácil leitura)
- [ ] Desativar notificações de plugins
```

#### Passo 5: Configurar Celular (CRÍTICO)

```bash
# PROTOCOLO DE CELULAR (faça SEMPRE)

## Passo 1: Modo Avião
Settings → Modo Avião: ON

## Passo 2: Esconder
- Gaveta
- Outro cômodo
- Mochila (fechada)

## Passo 3: NOTIFICAÇÕES (permanente)
Settings → Notificações:
- Desativar TUDO (exceto ligações de emergência)
- Desativar badges de app
- Desativar banners
- Desativar sons

## Passo 4: Apps Distratores (permanente)
- Deletar ou mover para pasta "Distrações"
- Usar app de bloqueio (Freedom, Cold Turkey)
- Modo cinza (torna apps menos atrativos)
```

**IMPORTANTE**: Celular na mesa, MESMO DESLIGADO, reduz foco em 10%.

#### Passo 6: Bloqueadores Digitais

```bash
# FERRAMENTAS RECOMENDADAS

## Desktop
# Freedom (cross-platform) - bloqueia sites/apps
freedom --start --duration 120m --blocklists social,news

# Cold Turkey (Windows/Mac)
# SelfControl (Mac)
# LeechBlock (Firefox)
# StayFocusd (Chrome)

## Sites para Bloquear (padrão)
- reddit.com
- twitter.com/x.com
- instagram.com
- facebook.com
- youtube.com (exceto se estudando com vídeo)
- news sites

## Comando customizado (Linux)
# Bloquear sites no /etc/hosts durante estudo
sudo block-sites.sh --enable
# Desbloquear depois
sudo block-sites.sh --disable
```

#### Passo 7: Preparar Recursos Físicos

```markdown
# ANTES de começar, garantir:

## Hidratação
- [ ] Copo/botão de água cheio na mesa
- [ ] Não precisar levantar durante foco

## Fome

## Banheiro
- [ ] Usar antes de começar

- [ ] Roupa adequada

## Postura
- [ ] Cadeira regulada
- [ ] Tela na altura dos olhos
- [ ] Pés no chão ou apoio
```

## 🎯 Framework 3D

### Onde Environment Design se Encaixa

| Dimensão | Como Environment Design Ajuda |
|----------|-------------------------------|
| **Conceitos** | Elimina distrações para foco profundo no #feynman |
| **Fatos** | Ambiente calmo para revisão SRS |
| **Procedimentos** | Setup pronto para #drill sem interrupções |

### Hierarquia de Foco

```
FOCO EFETIVO (pirâmide de necessidades):

         ┌─────────────┐
         │   Tarefa    │  ← O que você vai fazer
         │   Clara     │
         ├─────────────┤
         │  Técnica    │  ← Como fazer (Pomodoro, Feynman)
         │  Correta    │
         ├─────────────┤
         │  Ambiente   │  ← Base: SEM isso, resto não funciona
         │  Preparado  │
         └─────────────┘

Sem ambiente preparado:
- Não consegue aplicar técnica
- Gasta energia lutando contra distrações
- Cansa sem aprender
```

## ✍️ Boas Práticas

### ✅ BOM: Ritual de Setup (5 min fixos)

```bash
#!/bin/bash
# ritual-setup.sh - Script de preparação

echo "🎯 Iniciando ritual de Environment Design..."

# 1. Fechar abas
killall firefox 2>/dev/null || true  # ou salvar sessão antes

# 2. Abrir só o necessário
code . &                              # VS Code
obsidian &                            # Notas

# 3. Celular
echo "📱 Coloque o celular em modo avião e na gaveta"

# 4. Mesa
echo "🧹 Limpe a mesa (só computador e água)"

# 5. Notificações
echo "🔕 Ative modo Não Perturbe no computador"

# 6. Porta
echo "🚪 Feche a porta e coloque aviso"

echo "✅ Ambiente pronto! Inicie o Pomodoro."
```

```markdown
# USO: /ul-study-start OU ./ritual-setup.sh

10:00 | Ritual de Setup (5 min)
10:05 | Pomodoro 1 iniciado
```

### ❌ RUIM: Começar sem preparar

```markdown
# ❌ CENÁRIO COMUM (e ruim)

10:00 | "Vou estudar"
      | Abre laptop
      | 15 abas já abertas (YouTube, Reddit...)
      | Celular na mesa (notificações ligadas)
      | Porta aberta (pessoas passando)
      | Mesa bagunçada (papéis, comida)
      | Começa a ler
10:05 | Notificação no celular → Checa
10:07 | "Só vou responder isso"
10:15 | Volta, mas já perdeu foco
10:20 | Outra interrupção...

Resultado: 20 min de "estudo" = 5 min efetivos
```

### ✅ BOM: Zona de Foco Dedicada

```markdown
# ✅ CONFIGURAÇÃO IDEAL

## Espaco físico
- Mesa só para estudo/trabalho
- Nunca use para outras coisas
- Cérebro associa: "essa mesa = foco"

## Espaco digital
- Perfil de navegador "Estudos" (sem bookmarks de distração)
- Desktop workspace só para estudo
- Apps de estudo em tela cheia

## Ritual de entrada
1. Sentar na cadeira → Foco automático
2. Abrir IDE → Modo código
3. Fechar porta → Não perturbe

Após 2-3 semanas: Ambiente TRIGERA foco automaticamente
```

### ✅ BOM: Manutenção Semanal

```bash
# Toda sexta-feira: Revisar ambiente

## Checklist de Manutenção
- [ ] Limpar mesa (física e digital)
- [ ] Revisar abas salvas (fechar antigas)
- [ ] Organizar downloads
- [ ] Atualizar bloqueadores
- [ ] Revisar notificações (algo voltou?)
- [ ] Verificar cadeira/postura
- [ ] Limpar tela/teclado

# Tempo: 15 min
# Benefício: Ambiente otimizado para próxima semana
```

## 🎓 Resumo: Regras de Ouro

### ✅ 5 Regras para Environment Design Efetivo

1. **Celular: Modo avião + escondido**
   - ✅ Na gaveta/outro cômodo
   - ❌ Nunca na mesa (mesmo desligado)

2. **Mesa minimalista**
   - ✅ Só o essencial (computador + água)
   - ❌ Sem clutter visual

3. **Navegador: Abas zero**
   - ✅ Fechar tudo antes de começar
   - ❌ Não "vou fechar depois"

4. **Ritual de 5 minutos**
   - ✅ Mesma sequência sempre
   - ❌ Não improvisar cada vez

5. **Sinalização "Não Perturbe"**
   - ✅ Porta fechada + aviso
   - ❌ Assumir que ninguém vai interromper

## 📝 Exemplos Completos

### Exemplo 1: Setup de Estudo em Casa

#### Contexto
Estudante trabalhando em casa, família presente.

#### Problemas
- Irmãos interrompendo
- Celular na mesa
- Múltiplas abas
- Mesa bagunçada

#### Solução: Environment Design

```markdown
# SETUP FÍSICO

## Local
- Quarto com porta (não sala)
- Mesa pequena só para laptop
- Cadeira de jantar (temporária)

## Sinalização
- Porta fechada = "não entre"
- Aviso na porta: "Estudando 10h-12h"
- Fones de ouvido (mesmo sem música)

## Família
- Conversa: "Quando porta fechada, só emergência"
- Definir: Emergência = fogo, sangue, ou perigo
- Não emergência = "onde está X?", "olha isso"

# SETUP DIGITAL

## Celular
- Modo avião às 9:55
- Deixar no carregador (cozinha)
- Não pegar até 12:00

## Computador
- Script `study-mode.sh`:
  - Fecha todas as abas
  - Abre Notion + VS Code
  - Ativa Do Not Disturb
  - Bloqueia sites (Freedom)

## Mesa
- Só: laptop, água, caderno
- Tudo mais: fora do alcance
```

#### Resultado
- Antes: 2 interrupções/hora, 30 min para focar
- Depois: 0 interrupções, 5 min para focar
- Produtividade: +150%

### Exemplo 2: Setup em Espaço Compartilhado

#### Contexto
Estudante em biblioteca/coworking.

#### Desafios
- Outras pessoas ao redor
- Ruído ambiente
- WiFi público (distrações)

#### Solução

```markdown
# ESTRATÉGIA PARA ESPAÇO COMPARTILHADO

## Posicionamento
- Canto ou parede (costas protegidas)
- Longe de entrada/porta
- Longe de mesas de grupo

## Barreiras Físicas
- Fones de ouvido (noise cancelling ideal)
- Mochila ao lado (cria "bolha" pessoal)
- Tela virada para parede (privacidade visual)

## WiFi Público
- Usar bloqueador local (Freedom/hosts file)
- Ou: tethering do celular (mais seguro)
- Desativar notificações de rede

## Ritual no Local
1. Chegar → Escolher posição estratégica
2. Setup (3 min): Fones, água, organizar
3. Celular: Mochila/fechado (não no bolso fácil)
4. Sinal para outros: Fones + foco na tela

## Emergências
- Se alguém interromper: gesto educado de "ocupado"
- Se ambiente ficar barulhento: mudar de lugar
- Lista "pensar depois" para ideias que surgem
```

#### Resultado
- Consegue focar mesmo em biblioteca lotada
- Fones = sinal de "não perturbe" universal
- 0 interrupções de estranhos

### Exemplo 3: Recuperação Após Ambiente Ruim

#### Contexto
Semanas estudando em ambiente ruim, produtividade baixa.

#### Audit do Problema
```markdown
## Problemas Identificados:
1. Celular na mesa (olhava a cada 10 min)
2. Mesa na sala (TV ligada ao fundo)
3. 20+ abas sempre abertas
4. Notificações do Slack/email ligadas
5. Sem rituais (começava "quando dava")

## Impacto:
- Foco real: ~30% do tempo
- Cansaço alto (luta constante contra distrações)
- Frustração com progresso lento
```

#### Plano de Recuperação

```markdown
# SEMANA 1: Mudanças Básicas
- [ ] Mover setup para quarto
- [ ] Celular: modo avião + gaveta (obrigatório)
- [ ] Fechar TODAS as abas ao começar
- [ ] Desativar notificações do computador

# SEMANA 2: Otimização
- [ ] Comprar fones de ouvido
- [ ] Criar script de setup
- [ ] Definir horário fixo de estudo
- [ ] Avisar família sobre novo setup

# SEMANA 3+: Consolidação
- [ ] Avaliar: produtividade melhorou?
- [ ] Ajustar: o que ainda falha?
- [ ] Manter: ritual tornou-se hábito?
```

#### Resultado (Após 3 semanas)
- Foco real: 30% → 80%
- Sessões de estudo: menos cansaço
- Satisfação: alta (progresso visível)
