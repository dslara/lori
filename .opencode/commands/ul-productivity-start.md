## Uso
/ul-productivity-start

## Descrição

Quando não consegue começar a estudar, usa a Two-Minute Rule de James Clear (Atomic Habits): começar tão pequeno que é ridículo não fazer.

## Processo

### Passo 1: Identificar Resistência (2 min)

Perguntar o que está bloqueando:
```
"Vamos quebrar a inércia! Primeiro:

O que te impedindo de começar?

a) Tarefa parece grande/demorada
b) Não sei exatamente o que fazer
c) Não estou com vontade
d) Distrações (celular, rede social)
e) Outro: _______"
```

### Passo 2: Two-Minute Rule

Explicar a regra:
```
"🎯 TWO-MINUTE RULE

Pergunta: 'O que posso fazer em 2 minutos?'

O segredo: começar ridicularmente pequeno.

Exemplos:
❌ 'Vou estudar 1 hora de Go'
✅ 'Vou abrir o VS Code'

❌ 'Vou fazer 10 exercícios'
✅ 'Vou ler o enunciado do primeiro'

❌ 'Vou escrever código'
✅ 'Vou escrever 1 linha de comentário'

O cérebro resiste a tarefas grandes.
Aceita tarefas microscópicas.
Depois que começa, momentum leva adiante."
```

### Passo 3: Definir Micro-Tarefa (2 min)

Criar tarefa de 2 minutos específica:
```
"Vamos definir SUA micro-tarefa:

Qual seu objetivo de hoje? (mesmo que vago)
[Usuário responde]

Qual a versão de 2 minutos disso?

Exemplos:
• 'Estudar Go' → 'Abrir arquivo .go'
• 'Fazer projeto' → 'Criar pasta do projeto'
• 'Revisar código' → 'Ler 1 função'

Sua micro-tarefa de 2 min: ___________"
```

### Passo 4: Executar Sem Compromisso (2 min)

**Instrução crítica:**
```
"⏱️ REGRA DE OURO:

Você só precisa fazer a micro-tarefa.
NÃO precisa continuar depois.

Permissão para parar após 2 minutos.
Sem culpa. Sem obrigação.

Mas você PODE continuar se quiser.

Pronto? Vamos lá!

2 MINUTOS COMEÇAM... AGORA! 🚀"
```

### Passo 5: Verificar Resultado (1 min)

Após 2 minutos:
```
"⏱️ 2 minutos passaram!

O que aconteceu?

a) ✅ Fiz só a micro-tarefa e parei
   → Ótimo! Objetivo cumprido.

b) 🔄 Fiz a micro-tarefa e continuei
   → Perfeito! Momentum funcionou.

c) ❌ Não consegui fazer nem a micro-tarefa
   → Vamos diminuir mais! Qual versão
      ainda menor dessa tarefa?

Se escolheu (b), continue até naturalmente
querer parar. Não force.
```

### Passo 6: Consolidação

```
✅ Two-Minute Rule Aplicada!

📊 Resultado:
• Micro-tarefa: [tarefa]
• Tempo real: [X] minutos
• Conseguiu: [sim/não]
• Continuou: [sim/não]

💡 Insight:
A ação gera motivação, não o contrário.
Esperar vontade = procrastinar.
Começar pequeno = momentum.

🎯 Próxima vez:
Sempre que resistir, pergunte:
'O que posso fazer em 2 minutos?'
```

```
"Perfeito! Isso é normal.

🎯 TWO-MINUTE RULE

A vontade NÃO vem antes da ação.
Ação gera vontade.

Vamos começar RIDICULAMENTE pequeno.

O que você precisa fazer hoje?
(grande ou pequeno, não importa)"

Usuário: "Estudar algoritmos de ordenação"

"Bom objetivo! Agora vamos encolher:

Versões de 2 minutos:
❌ 'Estudar algoritmos' (grande)
❌ 'Ler sobre quicksort' (ainda grande)
✅ 'Abrir o arquivo sobre quicksort' (micro!)
✅ 'Ler o título da primeira seção' (micro!)
✅ 'Escrever 'quicksort' num papel' (micro!)

Sistema:

⏱️ REGRA DE OURO:

Pode fechar imediatamente depois.
Sem culpa.

MAS se quiser continuar, pode.

2 MINUTOS... VALENDO!
🚀🚀🚀"

[2 minutos depois]

Sistema:
"⏱️ Tempo!

O que aconteceu?

a) Abri e fechei (objetivo cumprido!)
b) Abri e continuei lendo (ótimo!)
c) Nem isso consegui"

Usuário: "b) Continuei lendo sobre quicksort"

Sistema:
"🎉 EXCELENTE! 🎉

O momentum funcionou!
Você começou com 2 minutos
e continuou naturalmente.

Isso acontece porque:
• Ação gera motivação
• Começar é o mais difícil
• Depois que começa, flui

📊 Resultado:
• Micro-tarefa: Abrir arquivo
• Tempo real: 15 minutos de estudo
• Conseguiu: ✅
• Continuou: ✅

→ /ul-study-start

```

## Quando Usar

✅ **USE quando:**
- Não consegue começar
- Está procrastinando
- Tarefa parece grande/demorada
- Não está com vontade
- Deixando para depois

❌ **NÃO USE quando:**
- Já está estudando (use /ul-study-end se quiser parar)
- Está no meio do fluxo
- Fazendo pausa planejada

**Fonte:** James Clear - Atomic Habits (Two-Minute Rule)

## Integrações

**Commands relacionados:**
- `/ul-study-start` — Se continuou estudando
- `/ul-productivity-break` — Se precisar de pausa
- `/ul-study-end` — Se formalizar como sessão

## Handoff

- Continuou estudando → `/ul-study-start`
- Precisa de pausa → `/ul-productivity-break`
- Quer encerrar → `/ul-study-end`
